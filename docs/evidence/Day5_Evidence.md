# Day5 Evidence - 状态管理与用户中心

## 1. 今日完成内容

今日主要完成了基于 Pinia 的状态管理系统搭建、用户中心页面开发以及全业务线收藏功能的实现。首先创建了 `userStore` 用于管理当前用户信息，在导航栏和发布页面中使用该状态；然后创建了 `favoriteStore` 用于管理收藏数据，为四个业务类型页面（二手交易、失物招领、拼单搭子、跑腿委托）添加了收藏操作；最后完善了个人中心页面，展示用户资料和收藏列表，并调整了收藏按钮的位置至卡片右下角。

## 2. Store 设计说明

| Store 文件 | 管理内容 | 主要状态 | 主要方法 |
|---|---|---|---|
| src/stores/user.ts | 当前用户信息 | isLoggedIn、currentUser | updateProfile、login、logout |
| src/stores/favorite.ts | 收藏状态 | favorites | addFavorite、removeFavorite、toggleFavorite、isFavorite |

**userStore 设计：**
- 使用 `defineStore('user', {...})` 定义用户状态 Store
- `currentUser` 对象包含 id、name、college、grade、avatar、bio 字段
- `displayName` getter 返回用户名称，`userDescription` 返回学院和年级组合
- `updateProfile` 方法支持部分更新用户信息，`login` 和 `logout` 方法控制登录状态

**favoriteStore 设计：**
- 使用 `defineStore('favorite', {...})` 定义收藏状态 Store
- `favorites` 数组存储收藏项，每项包含 id、type、title、description、location 字段
- `type` 字段区分四种业务类型：trade（二手交易）、lostFound（失物招领）、groupBuy（拼单搭子）、errand（跑腿委托）
- `isFavorite` 方法判断某项是否已收藏，`toggleFavorite` 方法切换收藏状态

## 3. 状态边界说明

**放入 Store 的数据：**

1. **当前用户信息**（放入 userStore）：因为导航栏需要显示用户名称和头像，发布页面需要将当前用户作为发布人，个人中心需要展示用户资料，这些跨页面共享的用户状态适合放入全局 Store。

2. **收藏列表**（放入 favoriteStore）：因为列表页（TradeView、ErrandView、GroupBuyView、LostFoundView）需要显示收藏状态和执行收藏操作，个人中心需要展示所有收藏内容，收藏数据需要在多个页面间同步，因此放入全局 Store。

**未放入 Store 的数据：**

1. **表单校验错误**：只属于发布页面的临时状态，不需要跨页面共享，因此使用页面内的局部状态管理。

2. **页面列表数据**（如 trades、errands 等）：虽然每个业务类型的列表数据在列表页使用，但不同页面的数据互不关联，且每次进入页面会重新请求，因此使用页面内的局部状态。

3. **详情页数据**：只在详情页展示，不需要跨页面共享，使用页面内的局部状态即可。

4. **消息中心状态**：虽然补充了基础状态，但消息数据主要在消息页面使用，且需要从后端获取，因此作为页面内状态管理。

## 4. 页面使用记录

| Store | 使用场景 |
|---|---|
| userStore | AppHeader.vue - 显示当前用户名称和头像；PublishView.vue - 使用当前用户作为发布人；UserCenterView.vue - 展示用户资料（名称、学院、年级、个性签名） |
| favoriteStore | TradeView.vue - 点击收藏/取消收藏；ErrandView.vue - 点击收藏/取消收藏；GroupBuyView.vue - 点击收藏/取消收藏；LostFoundView.vue - 点击收藏/取消收藏；UserCenterView.vue - 展示收藏列表并支持取消收藏 |

具体使用方式：

- **AppHeader.vue**：通过 `useUserStore()` 获取用户状态，在右上角显示用户名称和头像
- **PublishView.vue**：通过 `useUserStore()` 获取当前用户，提交表单时将 `publisher` 字段设置为当前用户名称
- **TradeView.vue / ErrandView.vue / GroupBuyView.vue / LostFoundView.vue**：通过 `useFavoriteStore()` 的 `toggleFavorite` 和 `isFavorite` 方法实现收藏功能，收藏按钮显示在卡片右下角
- **UserCenterView.vue**：同时使用两个 Store，展示用户资料和收藏列表，收藏列表支持取消收藏操作

## 8. 遇到的问题与解决方法

**问题 1：收藏按钮位置不对**

- **问题描述**：初始实现时收藏按钮放在 footer 插槽中，使用 `margin-left` 控制位置，但在不同卡片中显示不一致，且影响页面布局

- **解决方法**：在 ItemCard 组件中添加专用的 `#favorite` 插槽，使用 CSS `position: absolute` 将收藏按钮固定定位在卡片右下角（`bottom: 18px; right: 18px;`），卡片本身设置 `position: relative`，确保定位正确

**问题 2：收藏数据跨页面不同步**

- **问题描述**：在列表页收藏商品后，进入个人中心页面看不到收藏内容

- **解决方法**：检查发现是因为两个页面使用了不同的 Store 实例，确保所有页面都通过 `useFavoriteStore()` 获取同一个 Store 实例，Pinia 会自动管理单例，确保状态在整个应用中共享

**问题 3：详情页无法打开**

- **问题描述**：点击商品卡片时，由于收藏按钮和卡片点击事件冲突，无法正常跳转到详情页

- **解决方法**：在收藏按钮上添加 `@click.stop` 阻止事件冒泡，确保点击收藏按钮不会触发卡片的跳转事件

**问题 4：Store 导入路径错误**

- **问题描述**：在页面中导入 Store 时出现路径错误

- **解决方法**：检查导入路径是否正确，使用相对路径 `../stores/user` 和 `../stores/favorite`，确保文件结构正确

## 9. 今日反思

**Pinia 状态管理对多页面前端应用的作用：**

Pinia 作为 Vue 3 的官方状态管理库，为多页面前端应用提供了清晰的状态管理方案。通过将全局共享的数据（如用户信息、收藏列表）放入 Store，可以实现跨页面的数据同步和状态共享，避免了通过组件传递 props 或使用事件总线带来的复杂性。在校园集市项目中，Pinia 的优势明显体现：用户信息在导航栏、发布页和个人中心之间共享，收藏状态在四个业务列表页和个人中心之间同步，这些都得益于 Pinia 的集中式状态管理。同时，Pinia 的模块化设计使得状态可以按业务逻辑进行拆分（如 userStore 和 favoriteStore），提高了代码的可维护性和可读性。对于前端应用来说，良好的状态管理不仅能提升开发效率，还能确保用户体验的一致性，例如收藏操作在任意页面都能正确反映状态变化，用户无需刷新页面即可看到最新的收藏列表。
